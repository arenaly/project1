# AI Agent-as-a-Service (AaaS) MVP PRD

## 1. 제품 개요

고객이 목적에 맞는 가상 팀(마케팅/개발/전략)을 빠르게 구성하고,
AI 에이전트들이 역할 기반 협업으로 결과물을 생성하도록 하는 SaaS 플랫폼.

핵심 가치:
- 팀 빌딩 시간 단축
- 반복 업무 자동화
- 결과물 품질/일관성 향상
- 작업 과정과 비용의 가시성 확보

### 1.1 핵심 타겟 (Super Individual)
- 1인 창업자
- 실무 중심 개별 회사원(기획/리서치/보고서 작성 담당)

이 타겟에게 제품은 단순 자동화 도구가 아니라,
"가상의 동료"이자 "개인 전용 부서" 역할을 수행해야 한다.

---

## 2. MVP 범위 (권장: 마케팅 팀 먼저)

### 포함
1. 템플릿 기반 팀 생성
   - `Marketing Team` 템플릿 제공
   - 에이전트 예시: Strategist, Researcher, Copywriter, Reviewer
2. 목표 입력 및 작업 분해
   - Global Goal 입력
   - 시스템이 Local Task로 분해 후 담당자 할당
3. 기본 협업 토폴로지
   - 작성 → 리뷰 → 승인(고객) 3단계 워크플로우
4. 실행 가드레일
   - 최대 턴 수, 최대 시간, 최대 비용
   - 루프 감지 시 강제 종료 + 요약 보고
5. 실행 로그/비용 리포트
   - 에이전트별 메시지, 토큰, 비용 추적

6. 단톡방형 협업 UI
   - 사용자가 팀장처럼 `@에이전트` 멘션으로 지시
   - 에이전트 간 토론/중간보고/결과 공유를 대화형으로 노출

7. 프리셋 템플릿 UX
   - "마케팅 어벤져스 팀", "신사업 리서치 팀", "번역/로컬라이징 팀" 등
   - 클릭 한 번으로 팀 생성 + 권장 워크플로우 자동 세팅

### 제외(후속)
- 멀티 도메인(개발팀, 전략팀) 고도화
- SSO, 권한 고급 정책, 온프레미스
- 복잡한 외부 툴 마켓플레이스

## 3. 사용자 시나리오

1. 고객이 “콘텐츠 마케팅 팀” 템플릿 선택
2. 캠페인 목표(Global Goal) 입력
3. 시스템이 Local Task 생성 및 순서 배치
4. 에이전트들이 순차/병렬로 작업 수행
5. Reviewer 에이전트 검토
6. 고객 승인 후 최종 산출물 제공

### 3.1 회사원의 가상 TF팀 시나리오
1. 사용자가 "신사업 리서치 팀" 템플릿 선택
2. `@Tech Tracker 오늘 AI 웨어러블 최신 동향 정리해줘` 지시
3. `@Data Analyst 시장성/경쟁사 비교 분석` 지시
4. `@Report Writer 임원 보고용 1페이지 요약` 지시
5. 사용자는 논리 검토 및 최종 수정만 수행

### 3.2 1인 창업자 마케팅 부서 시나리오
1. 사용자가 "풀스택 마케팅 팀" 템플릿 선택
2. Trend Catcher가 타깃 커뮤니티 트렌드 수집
3. Copywriter가 채널별 카피 초안 생성
4. Editor가 톤앤매너/맞춤법 검수
5. 사용자가 최종 발행 여부만 결정

## 4. 기능 요구사항

## 4.1 Team Blueprint
- 팀 템플릿 생성/조회
- 에이전트 역할, 권한, 출력 포맷, 금지 규칙 정의

## 4.2 Goal & Task Orchestration
- Global Goal 입력
- Local Task 자동 분해
- 태스크 상태: `queued | running | blocked | done | failed`

## 4.3 Communication Topology
- `hierarchical`, `review-loop`, `human-gate` 지원
- 특정 단계에서 고객 승인 필수 옵션
- 단톡방형 지시 라우팅: `@agent`, `@team`, `@all`
- 멘션 미지정 시 PM 에이전트가 자동 태스크 라우팅

## 4.4 Guardrails
- max_turns, max_duration_sec, max_budget_usd
- 동일 주장 반복/상호반복 루프 감지
- 위험 응답(근거 부족/오류) 시 verifier 태스크 자동 삽입

## 4.5 Observability
- run/task/agent 단위 로그
- 모델/토큰/비용 저장
- 실패 원인 분류(타임아웃, 가드레일, 외부 API 오류 등)

## 4.6 Super Individual UX 원칙
- 설정 최소화: 고급 설정은 숨기고 기본값 제공
- 5분 내 첫 성과: 템플릿 선택 → 목표 입력 → 초안 산출
- 학습 없는 인터랙션: 메신저형 대화/멘션 인터페이스
- 의사결정 중심 설계: "작성"보다 "검토/승인"에 사용자 시간 집중

---

## 5. 비기능 요구사항
- 멀티테넌트 데이터 격리
- 99.5% 이상 API 가용성 목표(MVP)
- 실행 재시도/중단/재개 지원
- 감사 로그 및 변경 이력 보관

---

## 6. KPI
- Task Success Rate
- Human Override Rate
- Cost per Successful Task
- 평균 실행 시간
- 월간 활성 팀 수
